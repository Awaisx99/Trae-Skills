# Skill: Screenshot-Based Design Workflow (Iterative, Pixel-Perfect)

## Purpose
To create, modify, and refine web designs starting from a screenshot with an iterative, user-approval-based approach, while continuously learning and updating skills from each task.

## When to Use
- Converting a webpage screenshot to pixel-perfect HTML/CSS
- Making adjustments to an existing design from a screenshot
- Any task where you start from a visual image of a webpage

## Procedure

---

### Phase 1: Initial Analysis & Justification
1. **Carefully Analyze the Screenshot**:
   - Note colors (use color picker to get exact HEX/RGB values)
   - Identify typography (font families, sizes, weights, line heights)
   - Measure spacing (margins, padding, gaps between elements)
   - Map layout structure (containers, columns, sections)
   - List all interactive elements (buttons, links, forms)
   - Identify assets (images, icons, illustrations)

2. **Justify Instructions to User**:
   - Before coding, summarize your analysis and planned approach
   - Explain any assumptions you're making (e.g., "I'm assuming this section is 1200px wide on desktop")
   - Ask for clarification if anything is ambiguous in the screenshot

---

### Phase 2: Initial Pixel-Perfect HTML/CSS Build
1. **Create Semantic HTML Structure**:
   - Use appropriate HTML5 elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, etc.)
   - Structure the page logically
   - Add accessibility attributes (`alt` text, ARIA labels when needed)

2. **Write Precise CSS**:
   - Start with CSS variables for design tokens (colors, fonts, spacing)
   - Use Flexbox/Grid for layouts
   - Match exact dimensions, spacing, and colors from the screenshot
   - Make it responsive (mobile-first approach)
   - Optimize for performance

3. **Deliver Initial Version to User**:
   - Share the HTML/CSS files (or a live preview if possible)
   - Ask user to check and approve the design

---

### Phase 3: Iterative Refinement
1. **Check User Feedback**:
   - If user **approves** the design: Proceed to make the requested adjustments
   - If user **does NOT approve**: First make the design pixel-perfect and match the screenshot exactly, then proceed to adjustments

2. **Make Adjustments**:
   - Implement the requested changes precisely
   - Test the changes across breakpoints

3. **Deliver Updated Version & Ask for Approval**:
   - Share the revised files
   - Ask user to review and confirm if everything is good
   - If user is still not satisfied, iterate: improve the design/adjustments based on feedback, and repeat until approval

---

## Continuous Learning & Skill Improvement
**Crucial Rule**: For every design, animation, or workflow task, you **must**:
1. **Document Lessons Learned**: After the task is complete, note any new techniques, tools, or best practices discovered
2. **Update Relevant Skills**: If the task revealed a gap or improvement in an existing skill, add the new knowledge to that skill file as an "Extras" or "Crucial Learnings" section
3. **Maintain Consistency**: Use the improved approach in all future similar tasks

## Expected Outcome
A fully approved, pixel-perfect design that matches the user's requirements, with skills updated to reflect new knowledge gained.
